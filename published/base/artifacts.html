This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide.

SE base profil for patient

This is the base Patient profile to be used when profiling on Patient in a Swedish context
SE base profil for practitioner role

This is the base Practitioner Role profile to be used when profiling on Practitioner Role in a Swedish context
SE base profile for organization

This is the base Organization profile to be used when profiling on Organization in a Swedish context
SE base profile for practitioner

This is the base Practitioner profile to be used when profiling on Practitioner in a Swedish context

Structures: Data Type Profiles

These define constraints on FHIR data types for systems conforming to this implementation guide.

Profile for LMA number

This profile is to be used specifically for expressing an LMA number as an identifier
SEBaseHumanName

This profile provides the use of Swedish middlename and own family name as extensions and is used in the base Patient profile

Structures: Extension Definitions

These define constraints on FHIR data types for systems conforming to this implementation guide.

SE extension for official address types

This extension defines the official ValueSet to be used when expressing address type in accordance with the Swedish authorities
SE extension för explicit efternamn

The extension is used in the base profile for Swedish Patient to explicity express which part of the name is used as the family name.
SE extension för mellannamn

The extension is used in the base profile for Swedish Patient to express the middle name that a person born before 2017 can have

Terminology: Value Sets

These define sets of codes used by systems conforming to this implementation guide.

HSA valueset for business classification codes

Codes used to describe the type of care activities an entity can provide.
SE ValueSet for marital status

This is a value set of marital status types that include both the HL7 official codesystem and the ones specific from the Swedish authorities.
SE ValueSet for official address types

This is the ValueSet for address type in accordance with the Swedish authorities
SE ValueSet for preferred name

This is the ValueSet for a code used on the name that is the preferred name. I.e, the name that is the official name the person would like to be called by according to the national person registry
SE ValueSet security labels

This is the ValueSet for security labels in accordance with the swedish authorities
SKR ValueSet for clinical specialties

SKR ValueSet for clinical specialties
SKR ValueSet for occupations

SKR ValueSet for occupations
ValueSet for certification occupation from HOSP

Certified occupations as listed in the Register of Authorised Healthcare Professionals (HOSP)
ValueSet for occupation codes from the National Board of Health and Welfare (SOSNYK)

The code system for professional categories is developed to be used in digital documentation, for reporting to registers, etc.
Valueset for HSA occupational positions in healthcare

The code system is used in the data element 'paTitleCode' in the HSA catalogue

Terminology: Code Systems

These define new code systems used by systems conforming to this implementation guide.

SE CodeSystem for marital status

This is the CodeSystem for marital status in accordance with the Swedish authorities (Skatteverket Navet)
SE CodeSystem security labels

This is the CodeSystem for security labels in accordance with the swedish authorities
